\begin{abstract}
В данной статье представлено формальное описание операционной семантики выражений языка программирования Go с использованием языка спецификаций ABML. Предложенная онтологическая модель позволяет строго задать семантику базовых конструкций языка и может служить основой для разработки инструментов верификации программ.

В тексте статьи для визуального разделения двух используемых языков программирования применяется стиль с цветными боковыми полосами:
\begin{itemize}
    \item Многострочный код на языке \textbf{Go} выделяется \textit{светло-зелёным фоном} и \textit{тёмно-зелёной боковой полосой};
    \item Многострочный код на языке \textbf{ABML} выделяется \textit{белым фоном} и \textit{синей боковой полосой}.
\end{itemize}

\textbf{Ключевые слова:} операционная семантика, язык Go, язык ABML, формальные методы, онтологическое моделирование.
\end{abstract}

\section{Введение}
Формальное описание семантики языков программирования является фундаментальной задачей, лежащей в основе разработки компиляторов, статических анализаторов и инструментов верификации программ. Традиционные подходы к спецификации семантики, основанные на естественном языке, часто страдают от неоднозначности и неполноты, что затрудняет создание формально верифицируемых инструментов.

В данной работе рассматривается фрагмент языка программирования Go и его операционная семантика, формализованная с помощью языка спецификаций ABML (Agent-Based Modeling Language). Выбор языка Go обусловлен его возрастающей популярностью в промышленной разработке, особенно в области системного программирования и создания распределённых приложений. Язык ABML, в свою очередь, предоставляет средства для описания онтологий предметных областей и формализации правил перехода между состояниями.

При построении модели на фрагмент языка Go накладываются следующие ограничения, не снижающие общности подхода, но упрощающие формализацию:
\begin{enumerate}
    \item Использование обобщённых типов (дженериков) не допускается. Поскольку язык Go является строго типизированным, компилятор выполняет инстанцирование шаблонных типов на этапе компиляции, что позволяет заменить обобщённые конструкции конкретными экземплярами;
    \item Имена полей структур должны быть уникальными в пределах одной структуры. Это ограничение является естественным для языка Go и разрешимо на этапе предварительной обработки путём переименования конфликтующих полей.
\end{enumerate}

Основной вклад работы заключается в разработке онтологической модели базовых конструкций языка Go и формальном описании их операционной семантики. Предложенный подход может служить основой для создания инструментов статического анализа, верификации программ и автоматической генерации тестов.

\section{Онтология фрагмента языка Go}
Построение формальной модели семантики языка программирования требует предварительного создания онтологии — системы понятий, описывающих синтаксические конструкции и семантические сущности языка. В данном разделе на языке ABML вводятся типы объектов, соответствующие элементам фрагмента языка Go: переменным, функциям, операторам, декларациям и т.д.

Классификация и порядок изложения опираются на официальную спецификацию языка Go \cite{go-spec}. Большинство наименований моделируемых объектов соответствуют их названиям в спецификации для удобства восприятия и облегчения поиска дополнительной информации. В отдельных случаях, однако, некоторые сущности были признаны излишними с точки зрения моделирования их функциональности, а отдельные имена заменены на более точно отражающие семантику описываемых объектов.

\subsection{Имена}
Имена объектов в языке Go являются важнейшей категорией, связывающей синтаксические конструкции с семантическими сущностями. Ниже вводятся типы, моделирующие имена различных объектов программы. Для понимания приводимых определений необходимо знакомство с основными конструкциями языка ABML. Ключевое слово \ABMLInline{typedef} вводит новый тип, конструкция \ABMLInline{(uniont ...)} определяет объединение типов, а \ABMLInline{mot} (Model of Type) — тип объекта с именованными атрибутами.

\begin{ABMLCode}
(typedef "identifier" (uniont string))
(typedef "variable name" "identifier")
(typedef "field name" "identifier")
(typedef "function name" "identifier")
(typedef "method name" "identifier")
(typedef "interface name" "identifier")
(typedef "label name" "identifier")
(typedef "type name" "identifier")
\end{ABMLCode}

Тип \ABMLInline{"identifier"} представляет общее понятие имени объекта в программе. Остальные типы конкретизируют это понятие для различных категорий объектов: \ABMLInline{"variable name"} — для переменных, \ABMLInline{"field name"} — для полей структур, \ABMLInline{"function name"} — для функций, \ABMLInline{"method name"} — для методов, \ABMLInline{"interface name"} — для интерфейсов, \ABMLInline{"label name"} — для меток переходов, \ABMLInline{"type name"} — для имён типов.

\subsection{Значения и память}
Моделирование памяти и значений является основой для описания операционной семантики. В силу строгой типизации языка Go каждый объект программы имеет явно определённый тип, что позволяет однозначно сопоставлять значениям их типы.

\subsubsection{Ячейки памяти и значения}
\begin{ABMLCode}
(mot "location" :at "type" "type" :at "value" "Go value")
(typedef "Go value" (uniont "literal" "location"))
\end{ABMLCode}
Тип \ABMLInline{"location"} описывает ячейку памяти — абстракцию, связывающую тип хранимых значений с их текущим содержимым. Такое представление позволяет моделировать как простые переменные, так и составные объекты, размещённые в памяти. Тип \ABMLInline{"Go value"} объединяет все возможные значения языка Go, которые могут быть либо литералами (непосредственно заданными значениями), либо ссылками на ячейки памяти.

\subsubsection{Константные значения}
Константные значения, включая комплексные числа, вычисляются на этапе компиляции и моделируются следующим образом:
\begin{ABMLCode}
(typedef "constant" (uniont "true" "false" int real "complex constant" string symbol))
(mot "complex constant" :at "re" real :at "im" real)
\end{ABMLCode}

\subsubsection{Литералы}
Литералы представляют собой значения, которые могут быть непосредственно записаны в исходном коде программы. Для составных типов вводятся соответствующие литералы:
\begin{ABMLCode}
(typedef "literal" (uniont "constant" "composite lit" "function lit" "channel lit"))
(typedef "composite literal" (uniont "array lit" "slice lit" "struct lit" "map lit"))
(mot "array lit" :at "type" "array type" :at "value" (listt "Go value"))
(mot "slice lit" :at "type" "slice type" :at "value" (listt "Go value"))
(mot "struct lit" :at "type" "struct type" :at "value" (cot :amap "field name" "Go value"))
(mot "map lit" :at "type" "map type" :at "value" (cot :amap "Go value" "Go value"))
\end{ABMLCode}
Конструкция \ABMLInline{cot :amap ...} задаёт ассоциативный массив, сопоставляющий ключам (имена полей или значения) соответствующие значения элементов.

\subsubsection{Функциональные литералы}
Функциональные литералы описываются отдельно, поскольку функции в языке Go являются значениями первого класса:
\begin{ABMLCode}
(mot "function lit" :at "signature" "signature" :at "body" "block")
(mot "signature" :at "parameters" (listt "parameter decl") 
     :at "result" (uniont (listt "type") (listt "parameter decl")))
(typedef "parameter decl" (uniont "single param decl" "multi param decl"))
(mot "single param decl" :at "name" "parameter name" :at "type" "type")
(mot "multi param decl" :at "names" (listt "parameter name") :at "type" "type")
\end{ABMLCode}
Тип \ABMLInline{"function lit"} моделирует анонимные функции. Сигнатура функции \ABMLInline{"signature"} содержит список параметров и типы возвращаемых значений. Декларации параметров разделяются на одиночные (\ABMLInline{"single param decl"}) и множественные (\ABMLInline{"multi param decl"}). Множественные декларации возникают при объявлении нескольких параметров с общим типом, например, \GoInline{func(a, b, c int)}. В процессе предварительной обработки все множественные декларации преобразуются в последовательности одиночных для унификации последующей обработки.

\subsubsection{Каналы}
Для моделирования каналов — механизма межгорутинного взаимодействия — вводится тип \ABMLInline{"channel lit"}:
\begin{ABMLCode}
(mot "channel lit" :at "type" "channel type" :at "cap" nat :at "buffer" (listt "Go value"))
\end{ABMLCode}
Атрибут \ABMLInline{"cap"} задаёт ёмкость буфера канала, а \ABMLInline{"buffer"} представляет собой очередь сообщений.

\subsection{Типы данных}
Система типов языка Go разделяется на базовые (простые) и составные типы.

\subsubsection{Базовые типы}
Базовые типы включают логический, числовые, символьный и строковый:
\begin{ABMLCode}
(typedef "type" (uniont "base type" "composite type"))
(typedef "base type" (uniont "boolean type" "numeric type" "rune type" "string type"))
(typedef "boolean type" (enumt "bool"))
(typedef "numeric type" (uniont "int type" "float type" "complex type"))
(typedef "int type" (uniont "int" "int8" "int16" "int32" "int64" 
                         "uint" "uint8" "uint16" "uint32" "uint64" "uintptr" "byte"))
(typedef "float type" (enumt "float32" "float64"))
(typedef "complex type" (enumt "complex64" "complex128"))
(typedef "rune type" (enumt "rune"))
(typedef "string type" (enumt "string"))
\end{ABMLCode}
Конструкция \ABMLInline{enumt} задаёт перечислимый тип, значениями которого являются указанные константы.

\subsubsection{Составные типы}
Составные типы объединяют структуры данных, построенные из базовых или других составных типов:
\begin{ABMLCode}
(typedef "composite type" (uniont "array type" "slice type" "struct type" 
                               "pointer type" "function type" "interface type" 
                               "map type" "channel type"))
\end{ABMLCode}

\subsubsection{Массивы и срезы}
\begin{ABMLCode}
(mot "array type" :at "len" nat :at "element type" "type")
(mot "slice type" :at "element type" "type")
\end{ABMLCode}
Массивы имеют фиксированную длину, известную на этапе компиляции, в то время как срезы представляют собой динамические представления последовательных фрагментов массивов.

\subsubsection{Структуры и указатели}
Структуры объединяют набор полей с различными типами:
\begin{ABMLCode}
(mot "struct type" :at "fields" (cot :amap "field name" "type"))
(mot "pointer type" :at "type" "type")
\end{ABMLCode}

\subsubsection{Функциональные типы}
\begin{ABMLCode}
(mot "function type" :at "signature" "type signature")
(mot "type signature" :at "types" (listt "type") 
     :at "variadic type" "type" :at "result" (listt "type"))
\end{ABMLCode}
Тип \ABMLInline{"type signature"} описывает сигнатуру функции в терминах типов: список типов параметров \ABMLInline{"types"}, тип для вариативного параметра \ABMLInline{"variadic type"} (если присутствует) и список типов возвращаемых значений \ABMLInline{"result"}.

\subsubsection{Интерфейсы}
Интерфейсы задают множество методов, которые должны быть реализованы:
\begin{ABMLCode}
(mot "interface type" :at "elements" (cot :amap "method name" "signature"))
\end{ABMLCode}
Пример интерфейса на языке Go:
\begin{GoCode}
type A interface {
    f(n int)     // "method name" = `f`, "signature" = `(n int)`
}
\end{GoCode}

\subsubsection{Словари и каналы}
Словари (карты) и каналы завершают систему составных типов:
\begin{ABMLCode}
(mot "map type" :at "key type" "type" :at "element type" "type")
(mot "channel type" :at "direction" "direction" :at "type" "type")
(typedef "direction" (enumt "bidirectional" "send" "receive"))
\end{ABMLCode}
Направление канала \ABMLInline{"direction"} может быть двунаправленным (\ABMLInline{bidirectional}), только для отправки (\ABMLInline{send}) или только для приёма (\ABMLInline{receive}).

\subsection{Блоки}
Блоки определяют области видимости переменных и меток. В языке Go блок представляет собой последовательность операторов, заключённую в фигурные скобки.

\begin{ABMLCode}
(mot "block"
    :at "statements" (listt "statement")
    ; semantic attributes
    :at "variable location" (cot :amap "variable name" "location")
    :at "label position" (cot :amap "label" nat)
)
\end{ABMLCode}
Атрибут \ABMLInline{"statements"} содержит список операторов блока в модельном представлении. Семантические атрибуты \ABMLInline{"variable location"} и \ABMLInline{"label position"} служат для управления областями видимости. Атрибут \ABMLInline{"variable location"} сохраняет для каждой переменной её ячейку памяти до входа в блок, что позволяет корректно обрабатывать затенение (shadowing) имён переменных. Атрибут \ABMLInline{"label position"} сопоставляет меткам их позиции в блоке, обеспечивая корректную обработку операторов перехода.

\subsection{Декларации}
Декларации вводят новые объекты в программу.

\subsubsection{Общее понятие декларации}
Общее понятие декларации представлено типом \ABMLInline{"declaration"}:
\begin{ABMLCode}
(typedef "declaration" (uniont "type decl" "var decl" "function decl" 
                               "method decl" "interface decl"))
\end{ABMLCode}

\subsubsection{Декларация типов}
Декларация типа связывает имя с определением типа:
\begin{ABMLCode}
(mot "type decl" :at "name" "type name" :at "type" "type")
\end{ABMLCode}
Пример на языке Go:
\begin{GoCode}
type Point struct{ x bool }
\end{GoCode}

\subsubsection{Декларация переменных и констант}
\begin{ABMLCode}
(mot "const decl block" :at "declarations" (listt "var decl"))
(mot "var decl" :at "names" (listt "variable name") 
     :at "types" (listt "type") :at "values" (listt "expression"))
\end{ABMLCode}
Тип \ABMLInline{"const decl block"} моделирует блок деклараций констант, позволяющий использовать генератор \GoInline{iota}. Значение \GoInline{iota} представляет собой номер строки в блоке, отсчитываемый от нуля. Пример из спецификации языка Go:
\begin{GoCode}
const (
    a = 1 << iota  // a == 1  (iota == 0)
    b = 1 << iota  // b == 2  (iota == 1)
    c = 3          // c == 3  (iota == 2, unused)
    d = 1 << iota  // d == 8  (iota == 3)
)
\end{GoCode}

\subsubsection{Декларация функций и методов}
\begin{ABMLCode}
(mot "function decl" :at "name" "function name" 
     :at "signature" "signature" :at "body" "block")
(mot "method decl" :at "receiver" "parameter decl" 
     :at "name" "method name" :at "signature" "signature" 
     :at "body" "block")
\end{ABMLCode}
Метод отличается от функции наличием получателя (\ABMLInline{"receiver"}), который связывает метод с типом. Примеры:
\begin{GoCode}
func add(p *Point, q *Point) {
    p.x += q.x
}

func (p *Point) add(q *Point) {
    p.x += q.x
}
\end{GoCode}

\subsubsection{Декларация интерфейсов}
Декларация интерфейса связывает имя с типом интерфейса:
\begin{ABMLCode}
(mot "interface decl" :at "name" "interface name" :at "type" "interface type")
\end{ABMLCode}

\subsection{Выражения}
Выражения в языке Go — это конструкции, которые после вычисления возвращают значение.

\subsubsection{Общее понятие выражения}
Общий тип выражения представлен как объединение конкретных типов выражений:
\begin{ABMLCode}
(typedef "expression" (uniont "literal" "conversion" "method expr" 
                               "selector expr" "index expr" "slice expr" 
                               "type assertion" "function call" 
                               "unary expression" "binary expression"))
\end{ABMLCode}

\subsubsection{Приведение типов}
Приведение типов (conversion) позволяет явно преобразовывать значения из одного типа в другой:
\begin{ABMLCode}
(mot "conversion" :at "type" "type" :at "expression" "expression")
\end{ABMLCode}
Пример: \GoInline{float64(2)} преобразует целое число в число с плавающей точкой.

\subsubsection{Метод-выражения}
Метод-выражения (method expression) создают функции на основе методов, преобразуя получатель в первый параметр функции:
\begin{ABMLCode}
(mot "method expression" :at "receiver type" "type" :at "name" "method name")
\end{ABMLCode}

\subsubsection{Выбор поля и метода}
Выбор поля структуры или метода по экземпляру:
\begin{ABMLCode}
(typedef "selector expression" (uniont "selector struct" "selector method"))
(mot "selector struct" :at "struct" "expression" :at "field" "field name")
(mot "selector method" :at "method" "expression" :at "method" "method name")
\end{ABMLCode}
Примеры: \GoInline{person.Age} (выбор поля), \GoInline{file.Close()} (вызов метода).

\subsubsection{Индексирование и срезы}
Индексирование применимо к массивам, срезам, картам и строкам:
\begin{ABMLCode}
(mot "index expr" :at "arr" "expression" :at "index" "expression")
\end{ABMLCode}
Выражение среза (slice expression) создаёт новый срез из существующего массива, строки или среза:
\begin{ABMLCode}
(mot "slice expr" :at "arr" "expression" :at "low" "expression" 
     :at "high" "expression" :at "max" "expression")
\end{ABMLCode}
Параметры \ABMLInline{low}, \ABMLInline{high} и \ABMLInline{max} задают границы среза. Последний параметр может отсутствовать, в этом случае ёмкость среза определяется как \ABMLInline{len(arr) - low}.

\subsubsection{Проверка типа}
Проверка типа (type assertion) применяется к интерфейсным значениям:
\begin{ABMLCode}
(mot "type assertion" :at "interface" "expression" :at "type" "type")
\end{ABMLCode}
Пример:
\begin{GoCode}
var i interface{} = 3
num := i.(int) // num == 3, тип int
\end{GoCode}

\subsubsection{Вызов функции и операции с каналами}
Вызов функции и операция чтения из канала:
\begin{ABMLCode}
(mot "function call" :at "function" "expression" :at "arguments" (listt "expression"))
(mot "<-1" :at 1 "expression")
\end{ABMLCode}

\subsubsection{Унарные выражения}
Унарные алгебраические выражения описываются соответствующими типами:
\begin{ABMLCode}
(typedef "unary expression" (uniont "+1" "-1" "^1" "!1" "*1" "&1"))
(mot "+1" :at 1 "expression")  ; +x == x
(mot "-1" :at 1 "expression")  ; -x == -1 * x
(mot "^1" :at 1 "expression")  ; поразрядное дополнение
(mot "!1" :at 1 "expression")  ; логическое отрицание
(mot "*1" :at 1 "expression")  ; разыменование указателя
(mot "&1" :at 1 "expression")  ; взятие адреса
\end{ABMLCode}

\subsubsection{Бинарные выражения}
Бинарные алгебраические выражения:
\begin{ABMLCode}
(typedef "binary expression" (uniont "1||2" "1&&2" "1*2" "1/2" "1%2" "1<<2" "1>>2" 
                                    "1&2" "1&^2" "1+2" "1-2" "1|2" "1^2" 
                                    "1==2" "1!=2" "1<2" "1<=2" "1>2" "1>=2"))
(mot "1||2" :at 1 "expression" :at 2 "expression")
(mot "1&&2" :at 1 "expression" :at 2 "expression")
(mot "1*2" :at 1 "expression" :at 2 "expression")
(mot "1/2" :at 1 "expression" :at 2 "expression")
(mot "1%2" :at 1 "expression" :at 2 "expression")
(mot "1<<2" :at 1 "expression" :at 2 "expression")
(mot "1>>2" :at 1 "expression" :at 2 "expression")
(mot "1&2" :at 1 "expression" :at 2 "expression")
(mot "1&^2" :at 1 "expression" :at 2 "expression")
(mot "1+2" :at 1 "expression" :at 2 "expression")
(mot "1-2" :at 1 "expression" :at 2 "expression")
(mot "1|2" :at 1 "expression" :at 2 "expression")
(mot "1^2" :at 1 "expression" :at 2 "expression")
(mot "1==2" :at 1 "expression" :at 2 "expression")
(mot "1!=2" :at 1 "expression" :at 2 "expression")
(mot "1<2" :at 1 "expression" :at 2 "expression")
(mot "1<=2" :at 1 "expression" :at 2 "expression")
(mot "1>2" :at 1 "expression" :at 2 "expression")
(mot "1>=2" :at 1 "expression" :at 2 "expression")
\end{ABMLCode}
Семантика этих выражений соответствует стандартным определениям языка Go.

\subsection{Операторы}
Операторы (statements) в языке Go представляют собой конструкции, которые изменяют состояние программы, но не возвращают значений. В отличие от выражений, операторы выполняют действия: управляют потоком выполнения, осуществляют присваивания, организуют циклы и условные переходы. Общее понятие оператора в модели представлено типом \ABMLInline{"statement"}.

\begin{ABMLCode}
(typedef "statement" (uniont "declaration" "label stmt" "empty stmt" "1++" "1--" 
                               "assignment stmt" "if stmt" "switch stmt" "for stmt" 
                               "return stmt" "break stmt" "continue stmt" "goto stmt" 
                               "go stmt" "1<-2" "fallthrough stmt" "select stmt" "defer stmt"))
\end{ABMLCode}

\subsubsection{Помеченные операторы}
Помеченный оператор позволяет присвоить метку любому оператору программы. Метки используются для управления потоком выполнения с помощью операторов \GoInline{break}, \GoInline{continue} и \GoInline{goto}.
\begin{ABMLCode}
(mot "label stmt" :at "label" "label name" :at "statement" "statement")
\end{ABMLCode}
Пример использования метки в языке Go:
\begin{GoCode}
OuterLoop:
    for i := 0; i < 10; i++ {
        for j := 0; j < 10; j++ {
            if i*j > 50 {
                break OuterLoop
            }
        }
    }
\end{GoCode}
В данном примере метка \GoInline{OuterLoop} позволяет оператору \GoInline{break} выйти не только из внутреннего, но и из внешнего цикла.

\subsubsection{Пустой оператор}
Пустой оператор не выполняет никаких действий. Он используется в случаях, когда синтаксис требует наличия оператора, но никаких действий выполнять не требуется.
\begin{ABMLCode}
(typedef "empty stmt")
\end{ABMLCode}

\subsubsection{Операторы инкремента и декремента}
В языке Go операторы инкремента и декремента являются операторами, а не выражениями, что отличает Go от многих других языков программирования. Они могут использоваться только в постфиксной форме и не возвращают значения.
\begin{ABMLCode}
(mot "1++ stmt" :at "1" "expression")  ; x += 1
(mot "1-- stmt" :at "1" "expression")  ; x -= 1
\end{ABMLCode}
Примеры использования:
\begin{GoCode}
x++
y--
\end{GoCode}

\subsubsection{Операторы присваивания}
Операторы присваивания включают простое присваивание и составные присваивания с арифметическими и битовыми операциями. В языке Go допускается множественное присваивание, когда левая и правая части содержат одинаковое количество выражений.
\begin{ABMLCode}
(typedef "assignment stmt" (uniont "1=2" "1+=2" "1-=2" "1|=2" "1^=2" 
                                    "1*=2" "1/=2" "1%=2" "1<<=2" "1>>=2" 
                                    "1&=2" "1&^=2"))
(mot "1=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1+=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1-=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1|=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1^=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1*=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1/=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1%=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1<<=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1>>=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1&=2" :at 1 (listt "expression") :at 2 (listt "expression"))
(mot "1&^=2" :at 1 (listt "expression") :at 2 (listt "expression"))
\end{ABMLCode}
Примеры присваиваний:
\begin{GoCode}
x, y = y, x      // множественное присваивание
a += 10          // составное присваивание со сложением
b &= 0xFF        // составное присваивание с побитовым И
\end{GoCode}

\subsubsection{Условный оператор \texttt{if}}
Условный оператор \GoInline{if} может содержать необязательную инициализирующую часть (простой оператор), условие и ветви \GoInline{then} и \GoInline{else}. Инициализирующая часть выполняется до проверки условия и позволяет объявлять переменные, область видимости которых ограничена конструкцией \GoInline{if}.
\begin{ABMLCode}
(mot "if stmt" :at "init" "statement" :at "condition" "expression" 
     :at "then" "block" :at "else" (uniont "if statement" "block"))
\end{ABMLCode}
Пример использования оператора \GoInline{if} с инициализацией:
\begin{GoCode}
if x := f(); x < y {
    return x, true
} else if x > y {
    return y, true
}
\end{GoCode}
В данном примере переменная \GoInline{x} доступна только внутри конструкции \GoInline{if} и её ветвей.

\subsubsection{Оператор выбора \texttt{switch}}
Оператор \GoInline{switch} в языке Go существует в двух формах: переключатель по выражению и переключатель по типу. В отличие от многих других языков, ветви \GoInline{switch} в Go не проваливаются по умолчанию в следующую ветвь.
\begin{ABMLCode}
(typedef "switch stmt" (uniont "expr switch stmt" "type switch stmt"))
(mot "expr switch stmt" :at "init" "statement" :at "controlling expression" "expression" 
     :at "cases" (listt "expr case clause"))
(mot "expr case clause" :at "cases" (uniont (listt "expression") "default") 
     :at "statements" (listt "statement"))
(cot "default")
\end{ABMLCode}
Переключатель по выражению позволяет выбрать ветвь на основе значения управляющего выражения:
\begin{GoCode}
switch a {
case 0, 1, 2:
    f()
default:
    g()
}
\end{GoCode}
Переключатель по типу используется для проверки динамического типа значения интерфейса:
\begin{ABMLCode}
(mot "type switch stmt" :at "init" "statement" :at "guard" "type switch guard" 
     :at "cases" (listt "type case clause"))
(mot "type switch guard" :at "name" "type name" :at "variable" "expression")
(mot "type case clause" :at "cases" (uniont (listt "type") "default") 
     :at "statements" (listt "statement"))
\end{ABMLCode}
Пример переключателя по типу:
\begin{GoCode}
switch t := x.(type) {
case nil:
    fmt.Println("x is nil")  
default:
    fmt.Println("don't know the type")
}
\end{GoCode}

\subsubsection{Оператор цикла \texttt{for}}
В языке Go существует единственная циклическая конструкция — оператор \GoInline{for}, который может быть представлен в двух формах: классический цикл с условием и цикл с оператором \GoInline{range}.
\begin{ABMLCode}
(typedef "for statement" (uniont "for condition" "for range"))
(mot "for condition" :at "init" "statement" :at "condition" "expression" 
     :at "post" "statement" :at "body" "block")
(mot "for range" :at "names" (listt "variable name") 
     :at "arr" "expression" :at "body" "block")
\end{ABMLCode}
Классический цикл \GoInline{for} может использоваться как цикл со счётчиком, как цикл \GoInline{while} (если опущены инициализация и пост-действие), а также как бесконечный цикл (если опущено условие):

Цикл с оператором \GoInline{range} предназначен для итерации по элементам составных типов данных: массивов, срезов, строк, карт и каналов.

\subsubsection{Операторы передачи управления}
К операторам передачи управления относятся \GoInline{return}, \GoInline{break}, \GoInline{continue} и \GoInline{goto}. Эти операторы изменяют естественный поток выполнения программы.
\begin{ABMLCode}
(mot "return" :at "expressions" (listt "expression"))
(mot "break" :at "label" "label")
(mot "continue" :at "label" "label")
(mot "goto" :at "label" "label")
\end{ABMLCode}
Оператор \GoInline{return} завершает выполнение функции и возвращает указанные значения. В функциях, не возвращающих значений, оператор \GoInline{return} может использоваться без аргументов:
\begin{GoCode}
func add(a, b int) int {
    return a + b
}
\end{GoCode}
Операторы \GoInline{break} и \GoInline{continue} могут использоваться с метками для выхода из вложенных циклов:
\begin{GoCode}
OuterLoop:
    for {
        for {
            break OuterLoop
        }
    }
\end{GoCode}
Оператор \GoInline{goto} осуществляет безусловный переход к указанной метке. Его использование в Go ограничено: переход не может перепрыгивать через объявления переменных.

\subsubsection{Оператор \texttt{go}}
Оператор \GoInline{go} запускает функцию в отдельной горутине — независимом потоке управления, который выполняется конкурентно с другими горутинами.
\begin{ABMLCode}
(mot "go stmt" :at "body" "expression")
\end{ABMLCode}
Пример запуска горутины:
\begin{GoCode}
go Server()
go func(ch chan<- bool) {
    for {
        sleep(10)
        ch <- true
    }
}(c)
\end{GoCode}

\subsubsection{Оператор отправки в канал}
Оператор \GoInline{<-} используется для отправки данных в канал или чтения данных из канала. В модели оператор отправки представлен отдельным типом.
\begin{ABMLCode}
(mot "1<-2" :at 1 "expression")
\end{ABMLCode}
Пример отправки значения в канал:
\begin{GoCode}
ch <- 42
\end{GoCode}

\subsubsection{Оператор \texttt{select}}
Оператор \GoInline{select} позволяет горутине ожидать выполнения нескольких операций с каналами одновременно. Выполняется та ветвь, операция в которой готова к выполнению первой. Если готово несколько ветвей, выбор происходит псевдослучайным образом.
\begin{ABMLCode}
(mot "select stmt" :at "statement" (listt "common clause"))
(mot "common clause" :at "case" (uniont "send stmt" "receive = stmt" 
                                   "receive := stmt" "default") 
     :at "statements" (listt "statement"))
(mot "receive = stmt" :at "names" (listt "expression") :at "expression" "expression")
(mot "receive := stmt" :at "names" (listt "variable name") :at "expression" "expression")
\end{ABMLCode}
Пример использования \GoInline{select}:
\begin{GoCode}
ch1, ch2 := make(chan string), make(chan string)
go func() { time.Sleep(1 * time.Second); ch1 <- "данные из канала 1" }()
go func() { time.Sleep(2 * time.Second); ch2 <- "данные из канала 2" }()

select {
case msg1 := <-ch1:
    fmt.Println("Получено:", msg1)
case msg2 := <-ch2:
    fmt.Println("Получено:", msg2)
}
\end{GoCode}

\subsubsection{Оператор \texttt{fallthrough}}
Оператор \GoInline{fallthrough} передаёт управление следующей ветви в операторе \GoInline{switch}. В отличие от многих языков, где проваливание происходит по умолчанию, в Go оно должно быть явно указано с помощью \GoInline{fallthrough}.
\begin{ABMLCode}
(typedef "fallthrough" (enumt "fallthrough"))
\end{ABMLCode}
Пример использования:
\begin{GoCode}
switch {
case true:
    fmt.Print("true & ")
    fallthrough
case false:
    fmt.Print("false")
}
\end{GoCode}

\subsubsection{Оператор \texttt{defer}}
Оператор \GoInline{defer} откладывает выполнение функции до момента возврата из окружающей функции. Отложенные функции выполняются в порядке LIFO (последним пришёл — первым ушёл), что часто используется для освобождения ресурсов.
\begin{ABMLCode}
(mot "defer stmt" :at "expression" "expression")
\end{ABMLCode}
Пример использования \GoInline{defer}:
\begin{GoCode}
for i := 0; i <= 3; i++ {
    defer fmt.Print(i)  // prints 3 2 1 0
}
